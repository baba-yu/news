## Future

本レポートのニュースから導出した3点の予測。

1. **「AI SaaS 間の OAuth 横断信任」が最大のエンタープライズリスクに** — Vercel–Context.ai 事例は、**社員が個人で連携した AI Office SaaS が企業の Google Workspace 全体を侵害経路に変える** ことを示した。2026 Q3 までに、主要 SSO プロバイダ（Okta / Entra ID / Google Workspace）は **AI SaaS 向け OAuth スコープを "AI 拡張" カテゴリとして分離** し、管理者が「Allow All」既定を一括ブロックできる制御を標準化する。CASB / SSPM ベンダーも「AI 連携が閲覧する企業データ量」を独立メトリクスとしてレポート化する。
2. **エージェントランタイムはサンドボックス層が「OS カーネル級の実装課題」へ昇格** — OpenClaw CVE-2026-41329（heartbeat 継承の `senderIsOwner` 検証不備、CVSS 9.9）と SGLang CVE-2026-5760（Jinja2 SSTI、CVSS 9.8）は、AI エージェントのサンドボックス実装が **90年代の Web ブラウザと同じ「権限モデル定義の未成熟」** 段階にあることを可視化。2026 後半には OpenClaw / Hermes / Codex / Antigravity の各ランタイムが **seccomp-bpf 相当のケイパビリティ分離 + 監査ログ必須化** を競い合い、NemoClaw のような「外側から包む」アプローチがエンタープライズ調達要件として制度化する。
3. **画像／テキスト／エージェントのモーダリティ統合が 2026 の主戦場** — ChatGPT Images 2.0（ネイティブ推論、8 枚一貫生成）と Codex のコーディング外拡張（computer use / memory / automations）は、OpenAI が **「単一モデル＋専用 UI」から「単一モデル＋複数 UI の同一推論予算」へ舵を切った** ことを示す。Anthropic Claude Design、Google Antigravity、Hermes Agent の tool gateway 拡張も同方向。2026 後半にはユーザが「画像生成のために別モデルを呼ぶ」必要が消え、**Token budget は単一 / マルチモーダルで統合課金** されるのが標準になる。
