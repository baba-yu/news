### Local LLM Optimization (KV-Cache Compression)

- **Google TurboQuant (ICLR 2026, Apr 23–25 Rio; Research blog Apr 24–29 traction) — 3.5-bit-per-value KV-cache compression na katas ng FP16 sa LongBench / Needle-in-a-Haystack nang walang retraining; 6x compression, 8x faster attention** — Algorithm: MSE-driven Fast Walsh-Hadamard rotation + 3-bit Lloyd-Max quantization ng rotated cache. Targeted ang official Python implementation ng Google sa Q2 2026; community impls sa MLX + Triton + vLLM + SGLang ay umiikot na. [Google Research - TurboQuant: Redefining AI efficiency with extreme compression](https://research.google/blog/turboquant-redefining-ai-efficiency-with-extreme-compression/)

- **SGLang TurboQuant integration — PR #21419 in-tree (`--kv-cache-dtype turboquant`); follow-up #23134 fused-Triton-decode/extend kernel rewrite umabot ng 93–105% bf16 throughput vs ~1% sa original implementation** — First-class deployment knob para sa compressed-cache inference; may hot-path full-pool dequant bug ang original PR na nagpababa ng throughput, ang fix issue #23134 ay direktang nagba-basa ng packed 4-bit KV sa attention. [GitHub Issue - SGLang #23134 TurboQuant fused Triton kernels](https://github.com/sgl-project/sglang/issues/23134)

- **PolyKV (arxiv 2604.24971, Apr 27) — shared asymmetrically-compressed KV-pool para sa multi-agent LLM inference; int8 keys + TurboQuant-MSE 3-bit values via HF DynamicCache injection** — Sumusulat nang isang beses ng compressed cache at ini-inject sa N independent agent contexts. Pinapanatili ng asymmetric compression ang softmax stability (keys sa int8) habang mas pinipiga ang values (3-bit FWHT-rotated Lloyd-Max).
