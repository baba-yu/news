# ニュースレポート 2026-04-19

対象期間: 2026-04-16 〜 2026-04-19（3日間ウィンドウ）

## Headlines

- NVIDIA が GTC 2026 で OpenClaw を企業向けにラップした **NemoClaw** を発表。学習ループ重視の Hermes Agent と「幅 vs 深さ」の対立構図が鮮明化。
- **vLLM v0.18 / v0.19** が相次ぎリリース。gRPC サービング、GPU 高速化された投機的デコード、Gemma 4 Day-1 サポートを投入。
- **Bonsai 1-bit LLM** が WebGPU ブラウザ推論デモを展開（Apr 16）。`llama.cpp` も `b8838`（Apr 18）でリモートサーバ接続と検証可能推論サンプルを追加。
- **Claude Opus 4.7** が Amazon Bedrock で GA。並行して Anthropic の Claude Mythos Preview（脆弱性発見特化）が Bedrock で Gated Preview に。
- 4月 Patch Tuesday で **CVE-2026-33826**（Windows AD, CVSS 8.0）、**CVE-2026-32157**（RDP Client, 8.8）、**CVE-2026-33827**（Windows TCP/IP, 8.1）を含む計164件が修正。CIS が間接プロンプトインジェクションの急増を警告。

## Future

以下3点は本レポートのニュースから導出した予測。

1. **1-bit ネイティブ学習が主流の量子化代替に** — Bonsai 8B が1.15GB で実用品質を示したことで、4/8bit 事後量子化ではなく学習時から1-bit で設計する流れが、エッジ LLM の標準になる可能性。次の四半期で Qwen / Llama ファミリの 1-bit 派生が続くと見る。
2. **「Agent Registry」方式への収斂** — AWS Agent Registry、NemoClaw の監査ログ・ガードレール、Hermes Agent の自己学習スキルが同時発生。2026 後半にはエージェント成果物（スキル、ツール権限、監査トレース）を配布するレジストリ標準が MCP の隣に生えると予測。
3. **間接プロンプトインジェクションが CVE の主カテゴリに** — 成功攻撃 YoY +190%、CVE-2025-53773 型（PR 本文からの RCE）が再現しやすい。2026Q3 までに CVSS 8.0+ の LLM 連携 CVE がパッチ火曜日の常連カテゴリ化する。

## News

### LLM Workflow

#### OpenClaw / NemoClaw / Hermes Agents

- **NemoClaw（NVIDIA）** — GTC 2026 で発表。OpenClaw を監査ログ、アクセス制御、ガードレールで包む企業向けセキュリティスタック。OpenClaw のエコシステム規模（346K stars、44K skills、50+ プラットフォーム）を企業導入向けに正規化する位置づけ。 [The New Stack - Persistent AI Agents Compared](https://thenewstack.io/persistent-ai-agents-compared/)
- **Hermes Agent（Nous Research）** — 「the agent that grows with you」。スキルを自律生成・精緻化・セッション間で累積する閉ループ学習を軸に、ゼロ CVE・サーバレス $5/月 でポジショニング。対応プラットフォームは6つ（Telegram, Discord, Slack, WhatsApp, Signal, email）。 [GitHub - NousResearch/hermes-agent](https://github.com/nousresearch/hermes-agent)
- **構図の比較** — OpenClaw は「統合の幅」、Hermes は「学習の深さ」。OpenClaw にはネイティブなスキル学習層がなく毎回新規問題として扱う。 [Medium - OpenClaw vs Hermes Agent](https://medium.com/data-science-in-your-pocket/openclaw-vs-hermes-agent-5eca4021dfb1)

### Local LLM

#### Qwen

- **オープンウェイト Qwen3.6-35B-A3B** — コンシューマ向けノート PC でローカル実行可能。視覚推論タスクで Claude Opus 4.7 を上回る評価を得た例もある（2026-04-17 レビュー）。 [n1n.ai - Qwen3.6 vs Claude 4.7 Local LLM Review](https://explore.n1n.ai/blog/qwen3-6-vs-claude-4-7-local-llm-review-2026-04-17)

#### Bonsai（1-bit LLM / 最適化）

- **PrismML Bonsai 8B** — 全重みが -1/+1 のネイティブ 1-bit 学習。8B が 1.15GB、16bit 比 14倍小型、4-5倍のエネルギー効率。iPhone 17 Pro Max で 0.068 mWh/tok。 [PrismML - Announcing 1-bit Bonsai](https://prismml.com/news/bonsai-8b)
- **Bonsai 1.7B + WebGPU** — 290MB までサイズ圧縮、ブラウザ内 GPU 推論のデモが拡散（2026-04-16）。Ollama マルチエージェントと Gemma4 26B のトピックもまとめられている。 [n1n.ai - Local Inference Breakthrough 2026-04-16](https://explore.n1n.ai/blog/local-inference-breakthrough-webgpu-ollama-gemma4-2026-04-16)
