## Future

以下3点は本レポートのニュースから導出した予測。

1. **「GGUF 供給経路」がサプライチェーン攻撃の新しい主戦場** — SGLang CVE-2026-5760（GGUF 経由の Jinja2 SSTI で RCE）は「モデルファイル自体が実行可能コードのキャリア」である現実を可視化した。2026 Q3 までに Hugging Face / Ollama Library / ModelScope は **モデルカード + chat_template の署名検証／サンドボックス実行** を標準要件化し、Foundry Local / Bedrock / vLLM も GGUF ロード時に SSTI スキャンをデフォルト有効化する。
2. **「ハイパースケーラ × フロンティアラボの排他同盟」時代** — Amazon–Anthropic の 10 年 $100B コミットは、MS–OpenAI 提携に続く二つ目の「ハードウェア専用設計 × プラットフォーム独占」契約となる。2026 後半〜2027 前半で Google–（内製 Gemini）, Anthropic–AWS, OpenAI–MS, Meta–社内（Llama）, xAI–Oracle/NVDA の**5極体制**が固定化する。Trainium / TPU / MAIA / MI300X の設計差が「モデルの挙動差」に直結し、開発者は**バックエンドポータビリティ**（MCP のようなランタイム非依存層）を必須要件として再評価する。
3. **プロプライエタリ再膨張とオープンウェイトの地政学分断** — Qwen3.6-Max-Preview が「プロプライエタリのまま API のみ公開」に舵を切った点、Anthropic も Mythos Preview で段階ゲーティング化した点から、2026 Q3 には**「最強モデル＝非オープンウェイト」が明確化**。一方、オープンウェイトは 35B-A3B / Bonsai / Llama 系に最適化が集中し、ローカル = 実用帯、クラウド = フロンティアという線引きが制度化される。

### AI Security

- **SGLang CVE-2026-5760（CVSS 9.8）** — GGUF の `tokenizer.chat_template` に Jinja2 SSTI を仕込み、`/v1/rerank` 呼び出し時に RCE。Qwen3 リランカ検出ロジックを活用したトリガ設計。`ImmutableSandboxedEnvironment` 移行で緩和。 [CERT/CC VU#915947 - SGLang RCE](https://kb.cert.org/vuls/id/915947)
- **Indirect Prompt Injection（IPI）の体系化**（Lakera 研究、Apr 20 公開）— Web ページ／PDF／MCP メタデータ／RAG ドキュメント／メール／メモリ／コードなど **AI システムが取り込むデータ全面** が攻撃面。信頼入力と非信頼入力を同一コンテキストに混在させる構造そのものが根本原因。 [Lakera - Indirect Prompt Injection](https://www.lakera.ai/blog/indirect-prompt-injection)
- **CIS Report（Apr 1、ウィンドウ外だが参照継続）** — プロンプトインジェクションを LLM01 の最上位リスクに位置付け、実被害例として資格情報窃取・不正操作・持続感染を列挙。 [CIS - New CIS Report Warns Prompt Injection](https://www.cisecurity.org/about-us/media/press-release/new-cis-report-warns-prompt-injection-attacks-pose-growing-risk-to-generative-ai)
- **GitHub Actions 経由の間接プロンプトインジェクション PoC**（継続）— Anthropic Claude Code Security Review、Gemini CLI Action、GitHub Copilot Agent の 3 製品で GITHUB_TOKEN 窃取の PoC。CVE 発行なしで静かに修正。 [SecurityWeek - Agents Vulnerable to Prompt Injection via Comments](https://www.securityweek.com/claude-code-gemini-cli-github-copilot-agents-vulnerable-to-prompt-injection-via-comments/)
