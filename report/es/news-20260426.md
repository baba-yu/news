## Future

Tres predicciones derivadas de este informe.

1. **El "ciclo de alta frecuencia release-then-retract de los agentes de codificación de frontera" reconfigura los criterios de adopción empresarial para Q3 2026.** Las versiones Claude Code v2.1.119/v2.1.120 del 24-25 de abril vieron **8 regresiones simultáneas + un auto-update revertido**; `claude install 2.1.117` se convirtió en la "checklist de supervivencia" comunitaria como plantilla del SaaS Survival Guide. Para Q3 2026, Anthropic / Cursor / OpenAI Codex / GitHub Copilot Agent / Gemini CLI introducen un esquema formal **"Stable / Edge / Nightly" de tres pistas**, y los criterios de compras empresariales pasan de "adoptar lo último" a **"fijar la versión LTS N-2 con runtime de auto-bajada"**.
2. **"Día-0 inferencia + entrenamiento RL + descuento inmediato de API" se convierte en el estándar "lanzamiento de stack completo" para LLMs chinos.** DeepSeek V4 completó la sprint en sucesión — lanzamiento del 24 de abril + **SGLang/Miles Día-0 RL verificado** + 25 de abril **API al 75% off hasta el 5 de mayo** + reportaje de Bloomberg del 26 de abril sobre la integración Huawei Ascend. Los lanzamientos LLM de frontera evolucionan de las tres etapas **"pesos → benchmark → API"** a las cinco etapas **"pesos + Día-0 inferencia + Día-0 RL + Día-0 descuento de API + Día-0 optimización de hardware."** Qwen 3.5 / Kimi K2.7 / GLM-5.5 / MiniMax M2.8 / Tencent Hunyuan 3.0 siguen el mismo patrón de stack completo para Q3 2026.
3. **"El compromiso de cadena de suministro vía herramientas AI (patrón Context.ai → Vercel)" encabeza el OWASP LLM Top 10 para Q3 2026.** Tres en serie del 19-20 de abril (brecha Vercel / Context.ai: OAuth Allow All → toma de Google Workspace → enumeración de variables de entorno), 21 de abril (ataque Comment-and-Control: Claude Code / Gemini CLI / Copilot todos secuestrados vía comentarios de GitHub), 24 de abril (LMDeploy SSRF: ataque real detectado a 12h31m de la divulgación). El centro del vector de ataque específico para AI se desplazó de "CVEs individuales" a **"puntos de unión de la cadena de suministro (scopes OAuth / comentarios de GitHub / URLs de model loader)."** Para Q3 2026, **OWASP LLM Top 10 v2026** eleva "Supply Chain Compromise via AI Integration" al #1.
